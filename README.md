# **doda 求人情報自動収集ツール (Doda Job Listing Scraper)**

本プロジェクトは、Playwrightを活用して「doda.jp」の求人情報を効率的に収集・抽出する自動化ツールです。
反復的な求人確認作業を自動化し、独自の条件でフィルタリングを行うことで、データに基づいた効率的な求職活動を支援することを目的としています。

## 1. 概要 (Overview)
開発目的: 特定の検索条件に合致する求人情報のタイトルおよび詳細URLの一括収集
主な機能: ログインセッションの維持、ブラックリストによる除外フィルタリング、人間らしい動作のシミュレーション、CSV形式でのデータ出力

## 2. 主な特徴 (Key Features)
 | 機能 | 詳細説明 |
| :--- | :--- |
| セッション管理 | launch_persistent_context を使用し、ブラウザのプロファイル（Cookie・セッション）を維持。実行のたびにログインする手間を省き、効率的な収集を実現。 |
|高度なフィルタリング | BLACKLIST 設定により、特定のキーワードを含む求人を収集段階で自動除外。精度の高いリスト作成が可能。|
| 検知回避（安定性） | random.uniform を用いた動的な待機時間を設定。機械的なアクセスを抑制し、サイト側の負荷軽減とアクセス制限のリスクを回避。 |
| 動的コンテンツ対応 | Playwrightの wait_for_selector を活用し、JavaScriptによる動的なレンダリングにも対応した確実なデータ抽出。 |
| データの構造化 | 収集結果はタイムスタンプ付きのCSVファイルとして自動保存。データの蓄積と事後の分析が容易。 |
   
## 3. 技術スタック (Tech Stack)

言語: Python 3.10+

ライブラリ

Playwright: ブラウザ自動化および動的スクレイピング

python-dotenv: 環境変数による機密情報の管理

csv, os, time, random: 標準ライブラリによる効率的な処理

## 4. セキュリティと保守性 (Security & Maintenance)

機密情報の保護: サイトのURLやローカルパスなどの環境依存情報は .env ファイルに切り出し、
.gitignore を設定することでソースコード共有時の情報漏洩を防止しています。

メンテナンス性: 収集ロジックと保存ロジックを分離（モジュール化）し、将来的な対象サイトの仕様変更にも柔軟に対応できる設計としています。
